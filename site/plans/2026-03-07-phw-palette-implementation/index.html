    PHW-Aligned Accessible Mermaid Palette — Implementation Plan¶

    For Claude: REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

    Goal: Replace all pastel Mermaid fills across 9 docs pages with a six-role PHW-branded semantic palette; add extra.css for MkDocs theme colours and Mermaid dark-mode defaults.

    Architecture: Pure find-and-replace on style directives in Mermaid fences + one new CSS file + two lines changed in mkdocs.yml. No diagram structures change. No page text changes. The mkdocs build --strict command is the test at every step.

    Tech Stack: MkDocs Material 9.x, Mermaid.js, CSS custom properties

    Task 1: Create docs/stylesheets/extra.css¶

    Files: - Create: docs/stylesheets/extra.css

    Step 1: Create the file

    /* =============================================================================
   Public Health Wales — Corporate brand colours for MkDocs Material
   Primary: #c60158 (PHW Magenta)   Secondary: #325083 (PHW Navy)
   Source: phw.nhs.wales (March 2026)
   ============================================================================= */

/* Override MkDocs Material primary/accent with PHW brand colours */
:root {
  --md-primary-fg-color:        #c60158;
  --md-primary-fg-color--light: #df2076;
  --md-primary-fg-color--dark:  #9b0146;
  --md-accent-fg-color:         #325083;
  --md-accent-fg-color--transparent: #32508320;
}

/* Dark mode: use PHW Navy as accent so it's visible on dark backgrounds */
[data-md-color-scheme="slate"] {
  --md-accent-fg-color: #56b4e9;
}

/* =============================================================================
   Mermaid — default node colours for nodes WITHOUT explicit style directives
   Ensures un-styled nodes look right in both light and dark mode.
   ============================================================================= */

/* Force diagram background to match page background in dark mode */
[data-md-color-scheme="slate"] .mermaid svg {
  background: transparent;
}

/* Default node fill/stroke in dark mode (for un-styled nodes) */
[data-md-color-scheme="slate"] .mermaid .node rect,
[data-md-color-scheme="slate"] .mermaid .node circle,
[data-md-color-scheme="slate"] .mermaid .node ellipse,
[data-md-color-scheme="slate"] .mermaid .node polygon,
[data-md-color-scheme="slate"] .mermaid .node path {
  fill: #325083;
  stroke: #1e3a5f;
}

[data-md-color-scheme="slate"] .mermaid .node .label {
  color: #ffffff;
}

/* Subgraph labels in dark mode */
[data-md-color-scheme="slate"] .mermaid .cluster rect {
  fill: #1e1e2e;
  stroke: #325083;
}

    Step 2: Verify the file exists

    ls docs/stylesheets/extra.css
    Expected: file listed

    Step 3: Commit

    git add docs/stylesheets/extra.css
git commit -m "feat(styles): add PHW brand CSS variables and Mermaid dark mode defaults"
